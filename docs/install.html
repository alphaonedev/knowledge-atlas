Install

End-to-end setup for macOS, Linux, or Windows (WSL). Five minutes to first knowledge card.

1. Clone & install Python dependencies

git clone https://github.com/alphaonedev/knowledge-atlas.git
cd knowledge-atlas
python3 -m venv .venv             # optional but recommended
source .venv/bin/activate         # Windows: .venv\Scripts\activate
pip install -r requirements.txt

2. Configure your LLM provider

Knowledge Atlas can use either xAI Grok or Anthropic Claude. Pick one — or both; the system auto-detects whichever key is set.

Create a .env file in the project root:

# xAI (default)
XAI_API_KEY=sk-...

# or Anthropic
ANTHROPIC_API_KEY=sk-ant-...

The .env file is gitignored. The server also reads ~/.env if you keep keys globally.

Cost note. Extraction runs ~3K input / ~1K output tokens per video. ~$0.02–$0.04 per video. A typical 60-video channel runs about $2.

3. Initialize the sources registry

cp sources.json.example sources.json

sources.json is gitignored — what you index stays private.

4. Index your first expert

python3 add_source.py \
  --id my-expert \
  --name "Display Name" \
  --url "https://www.youtube.com/@channel" \
  --domain "what they teach" \
  --expertise "their unique angle"

This registers the source and fetches every video's transcript into sources/<id>/transcripts/.

Now extract knowledge cards:

python3 extract_knowledge.py --source my-expert

One JSON file per video appears in data/knowledge/, each containing 4–12 paraphrased cards.

Build the unified atlas:

python3 build_knowledge.py

5. Launch the dashboard

Knowledge Atlas ships with a lifecycle CLI (atlas.py) that manages the Flask service as a background daemon. Use it instead of running app.py directly:

python3 atlas.py start       # launch in background, writes PID + log
python3 atlas.py status      # PID, uptime, port, atlas content stats
python3 atlas.py logs        # tail data/atlas.log (-f to follow)
python3 atlas.py restart     # stop then start
python3 atlas.py stop        # SIGTERM → grace period → SIGKILL fallback

After start, open http://127.0.0.1:5179/. ⌘K to search. Click Index expert in the header to add more sources without using the CLI.

Manual mode. You can still run python3 app.py directly in the foreground for development; the CLI just wraps that with PID management, log capture, and clean shutdown.

Optional · macOS menu-bar / Windows system-tray icon

For a GUI affordance, there's a cross-platform tray app:

pip install pystray pillow
python3 atlas_tray.py
  • macOS: an "A" icon appears in your menu bar — click for the menu.
  • Windows: the same icon appears in your system tray — right-click for the menu.
  • Icon color reflects state in real time: green = running, red = stopped, amber = transitional.
  • Menu: Start · Restart · Stop · Open dashboard · View logs · Reveal data folder · Copy status · Quit tray.
  • Polls atlas state every 3s; macOS notifications fire on state changes.
  • Quit tray only quits the menu-bar app — it does not stop the atlas service.
Run on login. On macOS, create a LaunchAgent plist in ~/Library/LaunchAgents/ that runs python3 /path/to/atlas_tray.py. On Windows, drop a shortcut into shell:startup.

6. Wire in Claude Desktop

Edit ~/Library/Application Support/Claude/claude_desktop_config.json (macOS):

{
  "mcpServers": {
    "knowledge-atlas": {
      "command": "/absolute/path/to/python3",
      "args": ["/absolute/path/to/knowledge-atlas/mcp_server.py"]
    }
  }
}
Use the absolute path to python3 — Claude Desktop launches subprocesses with a stripped PATH. Find it with which python3.

Fully quit Claude Desktop (⌘Q from the menu bar) and relaunch. knowledge-atlas will appear in Local MCP servers with 10 tools.

Try:

List the sources in my knowledge atlas.
Teach me about [topic] using the knowledge-atlas tools.
What does my atlas say about [concept]? Cite the source videos.

7. Wire in Claude Code

claude mcp add knowledge-atlas --scope user -- \
  /absolute/path/to/python3 /absolute/path/to/knowledge-atlas/mcp_server.py
claude mcp list
# expect: knowledge-atlas: ... - ✓ Connected

8. Smoke test

# Flask alive?
curl -s http://127.0.0.1:5179/api/source | python3 -m json.tool

# MCP server starts cleanly under a stripped env (mimics Claude Desktop)?
env -i HOME="$HOME" PATH="/usr/bin:/bin" python3 -c "
import asyncio
from mcp.client.stdio import stdio_client, StdioServerParameters
from mcp.client.session import ClientSession

async def main():
    p = StdioServerParameters(command='python3', args=['mcp_server.py'])
    async with stdio_client(p) as (r, w):
        async with ClientSession(r, w) as s:
            await s.initialize()
            t = await s.list_tools()
            print(f'OK · {len(t.tools)} tools')

asyncio.run(main())
"
# expect: OK · 10 tools
